# Support

Use GitHub issues for support, bug reports, and feature requests:

- Bug report: include input, expected output, actual output, package version, and Node.js version.
- Data issue: include source reference, affected province/ward/mapping, and verification notes.
- Feature request: describe the migration or address-processing workflow you want to support.

For production migrations, review low-confidence and candidate-only results manually. The bundled sample data is not official administrative data.
